## 1. 总体定位

### 1.1 求职主策略

当前背景以 Java 后端为主，因此最优策略是：

1. **以 Java 业务后端作为基本盘**  
   目标是覆盖大多数后端岗位，保证面试稳定性和 offer 概率。

2. **以 AI Agent 工程化作为差异化能力**  
   不主打模型训练，而是主打“业务系统 + Agent + 工具调用 + RAG + 线上工程”的组合能力。

3. **以量化 QD 作为低投入、高质量备选方向**  
   优先投 Java/Python 平台类、交易基础设施类、风控/回测/数据平台类岗位；除非岗位明确匹配，否则不把 C++ 低延迟和复杂金融数学作为主战场。

### 1.2 时间投入建议

| 方向 | 建议投入比例 | 目标 |
|---|---:|---|
| Java 业务后端 | 60% - 70% | 主投方向，必须系统准备 |
| AI Agent 开发 | 20% - 30% | 差异化方向，准备一个可讲项目 |
| 量化 QD | 5% - 15% | 低配备选，聚焦算法、系统、性能、交易基础设施 |

### 1.3 优先级定义

| 优先级 | 含义 |
|---|---|
| P0 | 必须掌握，高频直接影响通过率 |
| P1 | 高频加分项，建议系统准备 |
| P2 | 定向岗位深入，时间不够可延后 |

## 3. 面试能力模型

### 3.1 每个知识点的标准答法

准备任何八股或项目问题时，都按以下结构组织：

1. **定义**：这个概念是什么。
2. **机制**：底层如何工作。
3. **场景**：什么时候用，解决什么问题。
4. **坑点**：常见误区、边界、失效场景。
5. **排查**：线上出问题怎么定位。
6. **权衡**：为什么选这个方案，不选其他方案。

示例：

- 问 Redis 分布式锁，不只答 `SET NX EX`，还要答锁续期、误删、Lua 解锁、RedLock 争议、业务幂等、数据库唯一约束兜底。
- 问 Spring 事务，不只答传播机制，还要答自调用失效、异常类型、代理机制、多数据源、异步线程事务不传播。
- 问 MQ 重复消费，不只答幂等，还要答业务唯一键、消费状态表、事务边界、重试与死信、消息堆积排查。

### 3.2 项目表达标准结构

每个核心项目至少准备以下内容：

1. **业务背景**：系统解决什么业务问题。
2. **个人角色**：自己负责哪些模块，边界是什么。
3. **系统架构**：核心服务、存储、缓存、MQ、外部依赖。
4. **核心链路**：用户请求如何流转，关键数据如何写入和读取。
5. **技术难点**：并发、一致性、性能、稳定性、成本、复杂业务规则。
6. **方案权衡**：为什么选当前方案，替代方案是什么。
7. **结果指标**：QPS、RT、错误率、吞吐、资源成本、业务收益。
8. **故障与复盘**：出过什么问题，怎么定位，如何改进。
9. **可扩展性**：如果流量涨 10 倍，如何演进。

至少准备：

- 1 个主项目：能讲 30 分钟。
- 1 个辅助项目：能讲 10 分钟。
- 1 个难点专题：例如缓存一致性、MQ 幂等、慢 SQL、接口降级、Agent 工具调用安全。

## 7. AI Agent 开发路线图

AI Agent 方向不应按“算法研究员”准备，而应按“LLM 应用工程 + 业务后端工程 + 工具系统工程”准备。

### 7.1 核心定位

目标画像：

- 能设计 Agent 后端架构。
- 能接入企业系统工具。
- 能构建 RAG 知识库。
- 能控制权限、审计和安全风险。
- 能做 tracing、eval、badcase 分析。
- 能把 Agent 部署成稳定服务。

不应主打：

- 从零训练大模型。
- 深度讲 Transformer 数学推导。
- 只展示 prompt demo。

### 12.2 排查回答模板

面对线上问题，按以下顺序回答：

1. **确认影响面**：哪些接口、哪些用户、哪个时间段、错误率和延迟变化。
2. **看监控**：QPS、RT、错误率、CPU、内存、GC、线程、连接池、DB、Redis、MQ。
3. **查日志**：错误日志、traceId、异常堆栈、关键业务字段。
4. **分层定位**：客户端、网关、应用、缓存、数据库、MQ、下游依赖。
5. **止血措施**：限流、降级、扩容、回滚、关闭开关、隔离故障依赖。
6. **根因分析**：代码、配置、数据、流量、依赖、资源。
7. **长期治理**：监控、告警、压测、容量、灰度、自动化回归。

## 18. 核心结论

最优准备路径是：

1. **先保 Java 后端基本盘**：Java、JUC、JVM、Spring、MySQL、Redis、MQ、分布式、OS、网络、系统设计、项目。
2. **再用 AI Agent 做差异化**：RAG、Tool Calling、Agent 编排、eval、trace、guardrails、工程化部署。
3. **最后低成本覆盖 QD**：算法、OS、网络、性能、交易基础设施。

主线判断：

- Java 后端决定面试下限。
- 项目表达决定面试可信度。
- 系统设计决定中高级岗位空间。
- AI Agent 决定差异化竞争力。
- QD 方向只在岗位匹配时加大投入。
